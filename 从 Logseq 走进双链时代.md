# 从 Logseq 走进双链时代

长期学习的过程中对笔记软件的折腾不少。最开始学习技术的时候是从纸笔开始，然后是有道笔记，然后是用 Typora 统一管理本地的 Markdown 笔记文件，然后又从 Typora 转移到 Obsidian。

但无论是纸笔记录，还是电子记录。我都遇到了以下阻碍：

- 一开始对笔记就进行分类是一种心理负担
	- 当我遇到某些观点或信息想记录时，但无法立即找到合适的归类方式，最终还是放弃了记录。这种难以分类的阻碍让我对记录本身产生负反馈。
- 笔记难以查找
	- 随着笔记数量越来越多，目录分类越来越大，想要找的对应的笔记内容很困难。
- 后期分类维护困难
	- 因为认知的不断成长，知识量越来越多且杂，记笔记时的分类一定会在后期重新分类的。在重新分类的时候，会发现一篇笔记可能同事属于不同的类别，这就造成了分类非常困难。

## Zettelkasten 和双链笔记

双链笔记其实已经和 Zettelkasten 基本没有什么关系了。对于我来说，我更倾向于借鉴 Zettelkasten，安排自己的笔记流。

Zettelkasten 的一些核心观念依然是有价值的。自下而上的记录、原子化的知识卡片、知识卡片之间形成链路发现新的洞见和理解。这些有价值的核心观念也可以应用到自己的笔记流中。

我在双链笔记里记录时，没有了 Fleeting Notes、Literature Notes、Permanent Notes 的概念。从 Zettelkasten 借鉴过来的部分，那就是 Daily Notes 作为 Fleeting Notes 和 Literature Notes 的结合，知识卡片做为 Permanent Notes 形成链路。

## 专门用语

*声明：这些专门用语是从 [双链时代的快速无压记录](https://www.yuque.com/deerain/gannbs/ffqk2e) 处学习而来的，我的双链笔记流都是从这位大佬习得的*

### 「块」

![](https://i.imgur.com/N0t91IK.png)

示例图中的小圆点代表的是一个块。它可以有父块，也可以有子块。按 Enter 键，就可以创建一个新的块。按 Enter 再 Tab 键，就可以创建一个子块。

### 「块引用」

![](https://i.imgur.com/ShcSlqf.png)

块能被引用。复制块引用链接 → 粘贴块引用链接，块就被引用了。点击块引用链接立即跳转到原始块。

### 「传递型双链」和「关联型双链」

![](https://i.imgur.com/07p8GKa.png)
![](https://i.imgur.com/BFlKyG3.png)
「传递型双链」和「关联型双链」语法都是 `[[主题]]`，例如示例中的 `[[英语/介词]]`、`[[介词短语]]`。

- 「传递型双链」形式：单独作为一个块。
	- 示例中的 `[[英语/介词]]` 就是传递型双链
- 「关联型双链」形式：出现在句子中。
	- 示例中的 `[[介词短语]]` 就是关联型双链。

「传递型双链」和「关联型双链」是等价的，只是使用的方式不一样。「传递型双链」独占一个块，「关联型双链」出现在句子中。

### 「标签」

![](https://i.imgur.com/8PMvUAE.png)

标签的语法是 `#主题`，例如示例中的 `#英语语法`。

## 理想的双链笔记软件

### 语法 `[[]]` 链接到主题

当在块中输入 `[[主题]]` 的时候，就会创建或者链接到 `主题` 这一个页面。
![](https://i.imgur.com/PbKUm1x.png)

在主题页底部能看到反链。
![](https://i.imgur.com/kWo4UmV.png)

### 主题可合并、随机、统一

- 合并：两个主题可合并为一个主题。
	- 背景：有时候会需要将两个主题合并。
- 随机：点击随机按钮，可以随机出现记录过的主题页。
	- 背景：笔记多了就很难被重新回顾，随机功能就像是[“沉寂在海底的翻涌的笔记”](https://www.yuque.com/deerain/gannbs/ffqk2e#de868e50)
- 统一： 使主题之间有父子关系。
	- 知识可以背细分为子类，这时候就需要有父主题和子主题了。例如  `[[英语]]` 是个大主题，大主题 `[[英语]]` 有`[[英语/说]]`、`[[英语/读]]`、`[[英语/写]]` 四个子主题。这时候在 `[[英语]]` 的页面下会显示这四个子主题。

### 在反链内容中能区分出「传递型双链」和「关联型双链」。

「传递型双链」：单独作为一行，格式为 `[[主题]]`

## 为什么用 Logseq

- 减少记录笔记的心里负担。减少记录的阻力。
- 日志功能。打破了我原先对笔记软件的印象。记录在日志里，只要标记好标签主题，就能在以后的主题页面中看到对应的引用和上下文。而不是像旧笔记那样慢慢地被遗忘。
- 是一个合格的双链笔记软件。
- 「见叶见树又见林」

双链笔记的笔记 — Roam Research（$165/year） 价格昂贵。

Logseq 特点：

- 是一个合格的大纲型双链笔记软件
- Journals 可作为 Fleeting Notes
- [PDF 阅读笔记功能优秀](https://www.bilibili.com/video/BV1DU4y1g7Bf?t=410.6)

这就是为什么我选择 Logseq 的原因。

## 我如何使用 Logseq 的
### 处理信息流

我的网络信息流来源有：公众号、Youtube、Twitter、微博。将信息流统一收录到稍后阅读软件。放到 daily routine 中去消化。安排专门的时间去消化和理解。

脑子里突然出现想法也是一种信息流。将这种信息流用自己最方便的笔记软件快速记录。安排专门的时间记录到 Logseq 中。
### 日常使用举例
#### 用 daily notes 的方式快速无压录入
%%我觉得可以拿矩阵去举例了%%

今天逛 Logseq 论坛看到一个关于配置的内容，于是我想记录下来，我的步骤是这样的

1. 打开 Logseq，按下快捷键 gj 跳转到 Journals 板块
2. 在 Journals 的写下：
  - [[Logseq/配置]]
    - `:ui/show-empty-bullets?` 变量： 显示或隐藏内容为空的 bullets
3. 等我哪天需要整理 Logseq 的时候， 从 Logseq 的反链中看到需要整理的，直接拖拽正文即可。(拖拽的方式：长按这个 Block 的圆点，拖拽到正文）

备注：[[Logseq/配置]]是一个主题，如果记录笔记时不知道写什么主题，那就用 [[inbox]]作为主题，日后可以针对[[inbox]] 进行整理。

### 何时整理

1. 给自己一个定期回顾的时间。
2. 页面里有大量的传递型双链

### 如何整理

反链内容里会有「传递型双链」、「关联型双链」。
- 对于传递型双链的内容，转移到正文中再整理。
- 对于关联型双链的内容，保留在反链内容里，无需整理。
（每一条笔记，无论是「传递型双链」还是「关联型双链」，都可以自由选择是否留在正文中。不要让自己太教条了。整理的过程需要灵活。）
